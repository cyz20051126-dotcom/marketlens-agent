# MarketLens AI 工作流 SOP

## 目标

把分散的公开资料转化为结构化竞品情报简报，并且每个结论都能追溯到证据行和来源链接。

## 输入

- 品牌范围：瑞幸、库迪、星巴克中国、蜜雪冰城、霸王茶姬、古茗、茶百道。
- 分析维度：价格、扩张、加盟、定位、风险。
- 公开来源：公司公告、招股书、年报、行业新闻、可信市场报道。

## 流程

1. 定义品牌范围和分析维度。
2. 使用 DuckDuckGo HTML 搜索收集公开来源。
3. 将资料拆成 claim 级别的证据行。
4. 校验必填字段：品牌、维度、主张、来源标题、来源 URL、来源类型、日期、摘录、置信度、复核状态。
5. 根据来源类型、摘录质量、URL 和复核状态计算置信度。
6. 生成品牌画像、定位矩阵数据和分章节简报。
7. 人工复核低置信度证据，并将弱来源标记为 `needs_review`。
8. 导出 Markdown 简报、JSON 数据和证据表。
9. 将生成结果加载到 React 研究工作台。
10. 用浏览器在桌面端和移动端检查展示效果。

## 如何迁移到其他行业

替换品牌列表、资料收集 query 和定位假设即可。证据 schema、评分规则和导出格式可以复用。比如可以迁移到美妆、新能源车、SaaS 竞品、本地消费连锁等方向。

## 质量规则

- 没有来源的结论不能进入简报。
- 不暗示使用公司内部数据。
- 低置信度来源必须显式标注为 `needs_review`。
- 每个简报章节必须列出支持证据 ID。
- 所有前端数据应能从 `data/evidence.csv` 重新生成。

## 面试表述

这个项目不是要假装做了一个完美爬虫，而是展示一个人机协同的 AI 研究工作流：AI 帮助搜索、抽取、归一化和初稿生成，人负责范围定义、证据复核和最终判断。
